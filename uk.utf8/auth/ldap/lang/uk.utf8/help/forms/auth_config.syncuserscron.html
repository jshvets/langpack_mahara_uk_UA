Synchronisation des utilisateurs avec cron

Sélectionnez cette option pour activer une tâche au niveau du cron qui va automatiquement créer et/ou mettre à jour les utilisateurs, en se basant sur les informations contenues sur le serveur LDAP. Par défaut, la tâche cron ne se lance qu'une seule fois par jour, à minuit (heure du serveur). Vous pouvez modifier ceci en éditant l'enregistrement dans la table « auth_cron » ou en utilisant le script en ligne de commande qui se trouve sur htdocs/auth/ldap/cli/sync_users.php si vous désirez lancer cette tâche cron à un autre moment ou lui adjoindre d'autres paramètres.

Cette option ne fonctionne que si le cron est activé. Consultez le guide d'installation pour les informations concernant la configuration du cron.
