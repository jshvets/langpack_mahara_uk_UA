Bypass index name

La plupart du temps ce paramètre doit être laissé vide.

Toutefois, si ce paramètre est renseigné, Mahara chargera les nouvelles données d'index dans l'index d'Elasticsearch tout en continuant de rechercher depuis l'index d'Elasticsearch stocké sous le nom indiqué dans la rubrique « nom d'index ».

Si vous désirez réindexer tout le site dans un nouvel index, sans toutefois arrêter le moteur de recherche durant ce travail, vous pouvez utiliser cette rubrique pour dériver la réindéxation lancée par le cron dans un nouvel index, puisque les demandes de recherche utiliseront l'ancien index.

Une fois la queue de recherche vide, copiez ce nom d'index dans la variable $cfg->plugin_search_elasticsearch_indexname dans votre fichier config.php.

Ce pramètre ne peut pas être modifié depuis l'interface web. Vous devez changer sa valeur dans le fichier config.php file avec la valeur suivante :

$cfg->plugin_search_elasticsearch_bypassindexname.
