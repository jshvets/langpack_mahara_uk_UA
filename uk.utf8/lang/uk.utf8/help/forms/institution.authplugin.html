Plug-in d'authentification

Vous pouvez choisir ici comment les utilisateurs d'une institution s'authentifie pour entrer sur Mahara. Chaque institution doit avoir au moins une méthode d'authentification des utilisateurs.

Le plug-in d'authentification interne utilise la base de donnée de Mahara pour valider les noms d'utilisateurs et les mots de passe. Les utilisateurs doivent alors avoir créé un compte sur Mahara pour que cette méthode fonctionne. Cela peut être effectué par un formulaire d'enregistrement, ou en ajoutant un utilisateur, ou en ajoutant des utilisateurs par CSV.

La méthode d'authentification acune permet à quiconque d'entrer sur Mahara sans validation. Les utilisateurs peuvent alors utiliser un nom d'utilisateur et un mode de passe quelconque pour entrer. Cette méthode ne devrait être utilisée que pour des périodes de test !

Les autres méthodes d'authentification utilisent généralement des sources externes, comme un serveur LDAP ou une application tierce distante. Elles possèdent leurs propres paramètres que vous pouvez configurer dans une fenêtre surgissante à l'ajout d'une nouvelle méthode d'authentification. Certaines méthodes peuvent être non active car elles requièrent des extensions PHP qui sont manquantes sur votre installation, ou car la lien vers la source externe d'authentification ne peut être établie. Par exemple, le plug-in d'authentification « Persona » nécessite que l’extension PHP cURL PHP soit installée qu'un lien en HTTPS puisse être fait vers verifier.login.persona.org.

Un paramètre que toutes les méthodes d'authentification normalement partage est celui qui consiste à permettre ou non aux utilisateurs de créer eux-mêmes leur compte depuis un formulaire (par opposition aux comptes créés préalablement par les administrateurs, avec lequel les utilisateurs peuvent se connecter).
